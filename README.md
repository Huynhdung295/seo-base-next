# 🚀 Next.js Landing Page Boilerplate

A **performance-first, SEO-ready** Next.js boilerplate for building landing pages. Optimized for **Vercel** deployment.

Built for the community. ❤️

## ⚙️ Tech Stack

| Category | Technology |
|---|---|
| **Framework** | Next.js 16 (App Router) + React 19 |
| **Language** | TypeScript (Strict Mode) |
| **Styling** | Tailwind CSS + PostCSS |
| **UI Components** | Shadcn UI (Headless) + Radix UI |
| **Icons** | Lucide React |
| **Animation** | Framer Motion |
| **Forms** | React Hook Form + Zod |
| **Database** | Supabase |
| **Analytics** | Vercel Analytics + Speed Insights + GTM |
| **SEO** | Dynamic Metadata + JSON-LD (schema-dts) + Auto Sitemap/Robots |
| **DX** | ESLint + Prettier + Husky + lint-staged |

## 🚀 Quick Start

### 1. Install dependencies

```bash
npm install
```

### 2. Setup environment variables

```bash
cp .env.example .env.local
```

Edit `.env.local` with your values:

```env
NEXT_PUBLIC_SITE_URL=https://your-domain.com
NEXT_PUBLIC_GTM_ID=GTM-XXXXXXX
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
```

> **💡 All env vars are managed in a single file:** `src/lib/env.ts`
> Import `{ env }` from `@/lib/env` — never use `process.env` directly.

### 3. Setup Shadcn UI (optional)

```bash
npx shadcn@latest init
```

Add components as needed:

```bash
npx shadcn@latest add button input dialog
```

### 4. Setup Husky (Git hooks)

```bash
npx husky init
```

### 5. Run development server

```bash
npm run dev
```

### 6. Delete unused file

```bash
# This file is no longer needed (sitemap/robots are auto-generated by Next.js)
del next-sitemap.config.js
```

## 📖 Key Features

### 📌 Centralized Environment Variables

All env vars are validated with Zod in one file — `src/lib/env.ts`:

```tsx
import { env } from "@/lib/env";

// ✅ Type-safe, validated, with defaults
console.log(env.NEXT_PUBLIC_SITE_URL);
console.log(env.NEXT_PUBLIC_GTM_ID);

// ❌ Don't do this — no validation, no type safety
// process.env.NEXT_PUBLIC_SITE_URL
```

### 🗺 Auto Sitemap & Robots

Served automatically by Next.js — no build step needed:

- `/sitemap.xml` → generated from `src/app/sitemap.ts`
- `/robots.txt` → generated from `src/app/robots.ts`

Just add your routes to `sitemap.ts` as you create new pages.

### 📊 SEO Metadata Engine

Use `constructMetadata()` in any page:

```tsx
// Static metadata
export const metadata = constructMetadata({
  title: "About Us",
  description: "Learn more about our company.",
  canonicalUrl: "/about",
});

// Dynamic metadata
export async function generateMetadata({ params }) {
  const data = await fetchData(params.slug);
  return constructMetadata({
    title: data.title,
    description: data.excerpt,
    image: data.ogImage,
    canonicalUrl: `/${params.slug}`,
  });
}
```

### ⚡ Data Fetching & Caching

```
┌─────────────────────────────────────────────────────────┐
│ revalidate: 3600  → ISR  (cached, re-fetches hourly)   │
│ revalidate: 0     → SSR  (always fresh, no cache)      │
│ revalidate: false → SSG  (cached forever until deploy)  │
└─────────────────────────────────────────────────────────┘
```

```tsx
import { fetchWithCache } from "@/lib/fetcher";

// ISR — default, re-fetches every hour
const posts = await fetchWithCache<Post[]>("/api/posts");

// SSG — cached forever
const config = await fetchWithCache<Config>("/api/config", {
  revalidate: false,
});

// SSR — always fresh
const user = await fetchWithCache<User>("/api/me", {
  revalidate: 0,
});
```

### 🎬 Scroll Animations

```tsx
"use client";
import { motion } from "framer-motion";
import { useScrollAnimation } from "@/hooks/useScrollAnimation";

function MySection() {
  const { ref, variants, animationState } = useScrollAnimation({
    preset: "slideUp", // fadeIn | slideUp | slideDown | slideLeft | slideRight | scaleIn
  });

  return (
    <motion.div
      ref={ref}
      initial="hidden"
      animate={animationState}
      variants={variants}
      transition={{ duration: 0.5, ease: "easeOut" }}
    >
      <h2>This slides up when scrolled into view</h2>
    </motion.div>
  );
}
```
